## Phase 1: 프로젝트 기반 구축

### 목표
개발 환경 세팅, DB 스키마 구축, 인증/권한 시스템 구현

### 작업 항목

| 작업 | 설명 | 상태 |
|------|------|------|
| 프로젝트 구조 설정 | Spring Boot 백엔드 프로젝트 생성 | 🔲 |
| 프론트엔드 설정 | React + TypeScript 프로젝트 생성 | 🔲 |
| DB 스키마 설계 | ERD 작성 및 테이블 생성 | 🔲 |
| 엔티티 클래스 구현 | JPA 엔티티 및 Repository | 🔲 |
| 인증 시스템 | JWT 기반 로그인/로그아웃 | 🔲 |
| 권한 관리 | 역할별 접근 제어 (멘토/멘티/기관/운영자) | 🔲 |
| API 기본 구조 | 공통 응답 포맷, 예외 처리 | 🔲 |
| API 문서화 | Swagger/OpenAPI 설정 | 🔲 |

### 산출물
- [ ] 백엔드 프로젝트 (Spring Boot)
- [ ] 프론트엔드 프로젝트 (React)
- [ ] DB 스키마 (ERD)
- [ ] 인증/권한 API

## 진행 규칙

### 브랜치 전략
```
main          - 프로덕션
develop       - 개발 통합
feature/*     - 기능 개발
hotfix/*      - 긴급 수정
```

### 커밋 컨벤션
```
feat: 새로운 기능
fix: 버그 수정
docs: 문서 수정
refactor: 리팩토링
test: 테스트 추가/수정
chore: 빌드, 설정 변경
```

### PR 규칙
- Phase 단위 또는 기능 단위로 PR 생성
- 최소 1명 리뷰 후 머지
